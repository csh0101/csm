# Codex 会话管理器 PRD 反推稿

## 1. 文档说明

本文档基于当前 `frontend/` 前端实现反推产品设计。前端已呈现完整的信息架构和交互意图，业务数据由本地 Rust 后端扫描和 API 提供；路径扫描、复制路径、标签增删、备注保存、回收站恢复等 MVP 能力已接入真实数据流，S3 归档仍属于后续扩展方向。

## 2. 产品定位

Codex 会话管理器是一个面向开发者的本地会话工作台，用于扫描、检索、预览、标注、清理、归档和恢复本地 Codex 会话文件。产品重点解决本地会话数量增长后难以查找、难以判断价值、难以安全清理的问题。

## 3. 目标用户

| 用户角色 | 核心诉求 | 典型场景 |
| --- | --- | --- |
| 开发者 / Codex 高频用户 | 快速找到历史会话、复制本地路径、清理过期上下文 | 在多个项目中查找旧讨论、复用方案、删除测试会话 |
| 项目维护者 / Tech Lead | 按标签或状态整理项目会话资产 | 按 Frontend、Backend、PRD、Architecture 等标签归档知识 |
| 工具管理员 / 运维角色 | 配置归档目标、保证删除可恢复 | 将删除会话归档到 S3，支持误删恢复 |

## 4. 产品目标

1. 降低本地 Codex 会话检索成本，让用户能通过路径、摘要、标签和状态快速定位目标会话。
2. 建立会话生命周期管理机制，区分活跃、僵尸、已归档/回收站会话。
3. 提供安全清理能力，删除前归档，删除后可在回收站恢复。
4. 支持中文/英文双语界面，覆盖个人开发者和团队内部工具场景。

## 5. 非目标范围

1. 不在 MVP 阶段实现多人协作权限、评论流或审批流。
2. 不在 MVP 阶段实现云端全文搜索服务，优先本地索引和客户端过滤。
3. 不在 MVP 阶段替代 Codex 本身的会话生成、编辑或执行能力。
4. 不把 S3 作为唯一存储来源，S3 仅作为删除归档与灾备目标。

## 7. 核心用户流程

### 7.1 扫描与导入

1. 用户在顶部输入本地 Codex 会话目录。
2. 点击「扫描路径」。
3. 系统读取目录下的会话文件，解析为 Session 列表。
4. 系统展示会话名称、摘要、标签、最后编辑时间、文件大小和状态。

### 7.2 浏览、过滤与搜索

1. 用户进入「资源管理器」查看所有非删除会话。
2. 用户可切换「最近使用」「僵尸会话」「未标记」「标签组」「回收站」。
3. 用户可在当前筛选结果内输入关键字，匹配会话名称、摘要或路径。
4. 列表顶部展示当前筛选名称和结果数量。

### 7.3 会话预览与整理

1. 用户点击表格行，右侧打开会话预览。
2. 系统展示会话名称、路径、Markdown 内容预览。
3. 用户可以复制本地路径、添加/移除标签、填写内部备注。
4. 用户可根据内容判断是否保留、标注或清理。

### 7.4 删除、归档与恢复

1. 用户在预览面板点击「删除并归档至 S3」。
2. 系统先将会话内容上传到归档目标，再将本地会话标记为 deleted 或移动到回收站。
3. 回收站仅展示已删除会话。
4. 用户可在回收站中点击「恢复会话」，会话恢复为 active 状态并重新进入资源管理器。

## 9. 业务规则

1. `all` 展示所有非 `deleted` 会话。
2. `deleted` 只展示 `status=deleted` 的会话。
3. `recent` 展示最后修改时间在最近 7 天内的非删除会话。
4. `stale` 展示 `status=stale` 或最后修改时间超过配置阈值的非删除会话，默认 15 天。
5. `unlabeled` 展示标签为空的非删除会话。
6. 标签组只统计非删除会话。
7. 侧边栏各筛选项计数必须与对应筛选结果一致，不能用活跃总数代替僵尸、未标记等专项计数。
8. 搜索在当前筛选集合内执行，不跨越回收站边界。
9. 搜索字段包括 `name`、`excerpt`、`path`，大小写不敏感。
10. 删除操作必须先完成归档或明确进入失败状态，不能静默丢失文件。
11. 恢复操作将会话状态恢复为 `active`。

## 10. 信息架构与页面

### 10.1 单页工作台

- 左侧侧边栏：产品标题、导航、智能过滤、标签组、回收站、语言偏好。
- 中间主列表：工作区路径、扫描按钮、搜索框、当前筛选统计、会话表格、分页。
- 右侧预览面板：会话内容、`codex resume` 恢复命令、标签、备注、复制路径、归档删除/恢复操作。

### 10.2 数据对象

Session 表示从本地文件扫描得到的会话主体信息。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | string | 管理器内部会话唯一标识，由文件路径稳定生成 |
| codexSessionId | string/null | Codex 原生会话 ID，用于生成 `codex resume <id>` 命令 |
| name | string | 会话名称 |
| excerpt | string | 列表摘要 |
| fullContent | string | Markdown 全量内容，可为空 |
| path | string | 本地文件路径 |
| labels | string[] | 标签列表 |
| lastModified | ISO string | 最后修改时间 |
| size | number | 文件大小，单位 bytes |
| status | active/stale/deleted | 会话生命周期状态 |

MetadataFile 表示管理器持久化的本地配置与用户侧元数据。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| workspacePath | string/null | 上次成功扫描的工作区路径 |
| staleAfterDays | number/null | 僵尸会话阈值，保存后覆盖环境变量默认值 |

SessionMeta 表示由管理器维护的用户侧元数据，避免重扫覆盖用户整理结果。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| sessionId | string | 关联 Session |
| labels | string[] | 用户维护标签，可覆盖或补充扫描标签 |
| notes | string | 用户内部备注 |
| statusOverride | active/stale/deleted/null | 用户操作产生的状态覆盖 |
| updatedAt | ISO string | 元数据最后更新时间 |

ArchiveRecord 表示删除归档审计记录。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| sessionId | string | 关联 Session |
| sourcePath | string | 删除前本地路径 |
| archiveProvider | string | 归档提供方，MVP 为 S3 |
| archiveUri | string | 归档对象地址 |
| archivedAt | ISO string | 归档完成时间 |
| checksum | string | 可选校验值，用于恢复校验 |

## 11. MVP 验收标准

1. 用户输入合法路径并扫描后，列表能展示真实本地会话。
2. 全部、最近、僵尸、未标记、标签、回收站筛选结果与业务规则一致。
3. 顶部结果数和侧边栏各筛选计数与真实筛选结果一致。
4. 搜索可在名称、摘要、路径中命中，且不破坏当前筛选边界。
5. 点击会话后，预览面板展示 Markdown 内容、路径、恢复命令、标签和操作按钮。
6. 复制路径或恢复命令写入系统剪贴板。
7. 列表可按最后修改时间、文件大小升降序排序。
8. 用户可在线调整僵尸阈值，保存后筛选和计数立即按新阈值刷新。
9. 删除会话前完成归档；归档失败时不改变本地状态并提示错误。
10. 回收站可恢复会话，恢复后会话回到资源管理器。
9. 标签和备注保存后，刷新页面仍可读取。
10. 中英文切换覆盖所有当前 UI 文案。
11. `npm run lint` 和 `npm run build` 通过。

## 13. 技术实现建议

1. 前端继续保留 React + Vite + Tailwind 的单页工作台结构，优先替换数据来源，不重写 UI。
2. 后端可利用项目已有 Express 依赖提供本地 API，封装扫描、读取、归档、标签/备注持久化能力。
3. 本地持久化建议使用轻量 JSON/SQLite：保存标签、备注、删除状态、归档记录，不直接改写原始会话内容。
4. 扫描逻辑应将“文件存在状态”和“用户管理状态”分开，避免重扫覆盖标签、备注、回收站状态。
5. S3 归档记录需保存 bucket、object key、etag、归档时间和原始本地路径，便于审计和恢复。
6. 大量会话场景下，前端应避免一次性渲染所有行，P1 引入分页或虚拟列表。

## 14. 风险与待确认问题

1. Codex 本地会话的真实文件结构、命名规则和内容格式需要确认。
2. “删除并归档至 S3”是强业务承诺，需明确归档失败、凭证缺失、网络失败时的状态机。
3. 本地会话可能包含敏感代码、密钥或客户信息，S3 归档必须有显式配置和清晰提示。
4. 重新扫描时如何处理已移动、已删除、已归档文件，需要定义冲突策略。
5. 标签和备注是只保存在管理器中，还是写回会话文件，需要产品和技术共同确认。
6. 当前前端有多选和分页入口，但缺少批量动作入口；需要确认 MVP 是否只做单条操作。
